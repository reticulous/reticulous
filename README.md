# reticulous — the buildable

`reticulous/reticulous` is just one component of the [**Reticulous**](https://github.com/reticulous) project: the **buildable**, the straddle that assembles a device image. It is not the whole system — the Reticulum stack, the radio/IP interfaces, and the messaging and browsing apps each live in their own straddles. For what Reticulous is, why Reticulum, and the full family of components, read the **[org README](https://github.com/reticulous)**.

## What this straddle does

The buildable carries no firmware or board HAL of its own — it is the assembly point. It ships the browser SPA, the LittleFS data image, and the LCD launcher icons, and it pulls in the whole reticulous straddle family plus the spangap platform (the base runtime, the IP/web stack, and the on-device LCD UI). You build it *with* a board, which supplies the pin maps, bring-up hooks, and hardware kconfig (flash size, PSRAM mode, display).

It also ships the on-device user manual (`esp-idf/data/webroot/WELCOME.md`), which overrides the viewer straddle's generic welcome page as the viewer's home document. It lives under `esp-idf/` (registered via `SPANGAP_EXTRA_DATA_DIRS` in `esp-idf/CMakeLists.txt`) because `data/webroot/` is wiped and regenerated by the SPA build.

Every component starts automatically when it is in the build — there is no hand-written init sequence. The whole entry point is generated by the build from each staged straddle's declared hooks (see [INTERNALS.md](INTERNALS.md)).

## Building

The `hw-*` board straddles don't decide what the device does — they only make a board usable. Build the buildable *with* one:

```sh
spangap build reticulous/reticulous --with spangap/hw-tdeck      # LilyGo T-Deck Plus (has a screen → on-device UI)
spangap build reticulous/reticulous --with spangap/hw-heltecv4   # Heltec WiFi LoRa 32 V4 (headless)
```

A board with a screen additionally installs the on-device LCD UI itself, so a screenless board builds headless with no extra flags. Leave off `--with` for a generic build that runs Reticulum over WiFi on any ESP32-S3 with PSRAM — browser and CLI access, but no mesh radio.

Primary target is the **LilyGo T-Deck Plus** (ESP32-S3FN16R8, 16 MB flash, 8 MB octal PSRAM, SX1262 LoRa, 320×240 LCD, QWERTY, trackball, GPS); the **Heltec WiFi LoRa 32 V4** is a headless secondary. PSRAM is required — the Heltec V3, which has none, does not qualify.

## Security

The design has had dedicated attention on security and is intended to be securable, but the code as it stands should not be relied on where a life depends on it being unhackable.
